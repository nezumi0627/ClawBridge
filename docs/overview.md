# 🚀 プロジェクト概要

**ClawBridge** は、OpenClaw クライアントと世界中の多様な AI リソースを繋ぐ、高機能な **AI ブリッジ・ゲートウェイ** です。

---

## 🎯 目的
AI の民主化と利用コストの削減を目指し、通常は個別に設定が必要な複数の無料/低コスト AI プロバイダを、唯一の「OpenAI 互換エンドポイント」として統合します。

### 主な役割
- **統一インターフェース**: あらゆるモデルに対し、`/v1/chat/completions` の標準形式でアクセス可能にします。
- **インテリジェント・ルーティング**: モデル名に基づいて、Pollinations, G4F, Gemini, Groq, Puter などの最適なバックエンドへリクエストを振り分けます。
- **レスポンス・正規化**: プロバイダごとに異なるレスポンス形式を整形し、不要なメタデータ（思考プロセスや広告）を除去します。

## 🏗️ システム構成
ClawBridge は Node.js ベースの軽量なコアサーバを中心に構築されており、必要に応じて重量級の処理（G4F等）をサブプロセスとして管理します。

- **コア層 (Node.js)**: API 受付、認証、ルーティング、レスポンス変換を担当。
- **プラグイン層 (JavaScript)**: 各プロバイダ（Gemini, Groq, Pollinations 等）との通信ロジック。
- **バックエンド層 (Python)**: G4F を介した深層的な無料リソース探索。

## 🌟 想定ユースケース
- **コストを抑えた開発**: 有料 API キーを消費せずに、最新の GPT-4 や Llama-3, Claude-3.7 系をテスト・運用したい。
- **マルチモデル・ボット**: Discord ボット等で、単一の設定のまま複数の AI モデルを切り替えて使いたい。
- **安定性の向上**: 1 つのプロバイダがダウンしても、他の無料リソースへ自動で切り替わるシステムを構築したい。
